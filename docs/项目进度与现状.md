## 三、技术栈与配置要点

- **后端**：FastAPI + Uvicorn，端口动态（默认 8100）；SQLite 路径 `data/warfare.db`
- **前端**：Vite 构建，端口动态（默认 5173）；API/WS 通过 proxy 或环境变量指向后端
- **Ollama**：qwen2.5:3b（FAST_MODEL）、qwen3-vl:8b（ANALYSIS_MODEL）；本机 11434
- **时区**：后端统一 UTC（iso_utc），前端解析带 Z/+00:00 的 ISO 字符串

---

## 四、已知限制与可选优化

- **爬虫**：部分源因 403/网络被替换为备用 RSS（如 Reuters→NYT、AP→Guardian）；RSSHub 依赖可能不稳定
- **实时航班**：主源 api.adsb.lol，备用 OpenSky（易 429）+ 演示数据；中东区域航班数有限
- **舰船**：当前为演示数据；真实 AIS 需配置 AISSTREAM_API_KEY
- **视频热点**：YouTube 嵌入受策略限制，部分需登录或无法自动播放；已做多源 fallback
- **云部署**：见 `云部署资源评估.md`；需 8GB+ GPU 才能获得与本机接近的 AI 体验

## 六、小结

当前项目**核心链路已闭环**：多源抓取 → AI 摘要与分类 → 战场报告生成 → 地图/情报/分析展示 → 手动与定时刷新 → WebSocket 推送。近期已完成新闻模型升级（qwen2.5:3b + 关键词坐标）、多年度 EOX 底图与 NASA GIBS 专题层、Header 卫星对比弹窗（纳坦兹/班达尔阿巴斯前后对比）、端口冲突处理、弹窗与时间戳等体验优化，以及云部署评估文档。后续可按「BoShi-System-Guide.html」或「frontend/public/system-guide.html」中的路线图做功能扩展或性能与稳定性优化。
